# Resume Vibe & Structure Guide

This guide ensures that any expansion of the resume follows the premium engineering aesthetic established in the template.

## 1. Top Basic Info (Header Style)
The header follows a "Balance and Professionalism" principle.
- **Name (ZH & EN)**: The primary font is bold and large (`48px`). The English name should be next to the Chinese name in a lighter weight (`400`) and smaller size (`32px`).
- **Role Title**: Gray color (`#666`), size `20px`. It should summarize the candidate's core identity (e.g., "AI Engineering / System Architecture").
- **Contact Grid**: 
  - Labels should be a consistent width (`75px`) and a light gray (`#aaa`).
  - Values should be dark (`#333`) and bold enough to be legible.
  - The grid should be 2 columns. If there's an odd number of items, adjust to keep the balance.

## 2. Content Sections Structure
Every section must have a `.section-title` with a `2px solid #333` bottom border.

### Core Skills
- **Skill Category**: Bold, width `100px`. Use high-level terms (e.g., "Full Stack", "AI Engine").
- **Skill List**: Use a semicolon-separated list. Use strong keywords (e.g., **Spring Boot 3**, **Rust**, **Agents**).

### Work Experience
- **Header**: Contains Company Name, Role (separated by a vertical bar `|`), and Date range aligned to the right.
- **Content**: 
  - Use `.job-point` to group logical achievements.
  - Each point should start with a bold title: `[Title]: [Description]`.
  - Description should use impact-oriented verbs: "Maintained", "Optimized", "Architected", "Scaled".
  - Mention specific metrics or technologies where possible.

## 3. Aesthetic Principles
- **Spacing**: Use consistent margins (`30px` between sections).
- **Colors**: Never use pure black/white for text. Use `#333` for primary and `#666` for secondary content.
- **Micro-detail**: Use thin lines (`1px solid #eee`) for internal grid separations if needed.
- **Tone**: Professional, authoritative, yet innovative. Focus on "Architecture", "Engineering Practice", and "Solving Complexity".

## 4. Expansion Vibe
When the user asks to "Add more detail about X," the AI should:
1.  **Analyze the Technical Core**: What was the hardest part? (Concurrency, Scale, AI Orchestration?)
2.  **Describe the 'How'**: What specific protocol or pattern was used (e.g., "Based on MCP", "Used Event-Driven Architecture")?
3.  **Ensure A4 Fit**: If the content exceeds one page, prioritize the most recent/impactful experiences.

## 5. Visual Excellence (HR Preference)
Refer to the [HR Favorite Resume Format](./hr-favorite-format.png) for visual inspiration. 
- **Key Characteristics**: High contrast between Name/Role and Body content, generous whitespace, and a clean, right-aligned profile photo.
- **Hierarchy**: Clear visual separation between experience blocks and educational background.
